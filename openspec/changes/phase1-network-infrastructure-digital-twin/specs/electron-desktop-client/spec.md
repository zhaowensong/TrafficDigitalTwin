## ADDED Requirements

### 需求:Electron项目结构
系统必须创建标准的 Electron 项目结构。

#### 场景:项目目录结构
- **当** 创建 Electron 客户端
- **那么** 必须包含以下文件结构：
  - `package.json` - 项目配置和依赖
  - `main.js` - Electron 主进程
  - `preload.js` - 预加载脚本（安全）
  - `src/` - 前端代码目录
    - `index.html` - 主页面
    - `script.js` - 前端逻辑
    - `style.css` - 样式文件

### 需求:前端代码复用
系统必须复用现有的前端代码，最小化改动。

#### 场景:复制前端代码
- **当** 初始化 Electron 项目
- **那么** 必须复制现有 index.html、script.js、style.css
- **并且** 保持现有功能和样式不变

#### 场景:API地址配置化
- **当** 修改 script.js
- **那么** 必须将 API_BASE 改为可配置
- **并且** 支持从配置文件或界面读取服务器地址
- **并且** 默认值为本地 `http://127.0.0.1:5000/api`

### 需求:主进程实现
系统必须实现 Electron 主进程，加载前端页面。

#### 场景:主进程功能
- **当** 启动 Electron 应用
- **那么** main.js 必须：
  - 创建浏览器窗口（800x600 或更大）
  - 加载本地 src/index.html
  - 处理窗口关闭事件
  - 支持开发者工具（开发模式）

#### 场景:安全配置
- **当** 配置 Electron 窗口
- **那么** 必须启用 contextIsolation
- **并且** 使用 preload.js 进行进程间通信

### 需求:服务器配置界面
系统必须提供服务器地址配置功能。

#### 场景:配置界面
- **当** 用户首次打开应用或点击设置
- **那么** 必须显示服务器地址输入框
- **并且** 支持保存配置到本地文件
- **并且** 应用重启后配置仍然有效

#### 场景:配置验证
- **当** 用户输入服务器地址
- **那么** 必须验证地址格式正确
- **并且** 测试连接是否可用
- **并且** 给出连接成功/失败提示

### 需求:打包生成exe
系统必须支持打包生成 Windows 可执行文件。

#### 场景:打包配置
- **当** 配置 package.json
- **那么** 必须包含 electron-builder 配置
- **并且** 支持生成 .exe 安装程序
- **并且** 支持生成便携版（portable）

#### 场景:打包输出
- **当** 执行打包命令
- **那么** 必须在 dist/ 目录生成安装文件
- **并且** 文件大小应合理（< 200MB）
- **并且** 安装后可直接运行，无需额外依赖

### 需求:C/S架构支持
桌面客户端必须支持连接远程后端服务器。

#### 场景:本地后端连接
- **当** 配置为 `http://127.0.0.1:5000/api`
- **那么** 必须能正常访问本地后端
- **并且** 所有功能正常工作

#### 场景:远程后端连接
- **当** 配置为 `http://服务器IP:7860/api`
- **那么** 必须能正常访问远程后端
- **并且** 数据加载和 AI 预测正常工作
- **并且** 地图和图表正常显示

### 需求:跨域处理
系统必须处理桌面客户端与后端服务器的跨域问题。

#### 场景:CORS配置
- **当** 桌面客户端请求后端 API
- **那么** Flask 后端必须允许跨域请求
- **并且** 或者 Electron 禁用 webSecurity（开发模式）

### 需求:离线提示
系统必须在无法连接后端时给出明确提示。

#### 场景:连接失败
- **当** 后端服务器无法连接
- **那么** 必须显示错误提示界面
- **并且** 提供检查网络或修改配置的选项
