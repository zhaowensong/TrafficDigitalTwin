## ADDED Requirements

### 需求:基站基础属性扩展
系统必须扩展基站数据，在现有 id 和 location 基础上增加基础属性。

#### 场景:基站数据结构
- **当** 加载基站数据
- **那么** 每个基站必须包含以下字段：
  - id: 字符串，基站唯一标识
  - name: 字符串，基站名称（如"基站001"）
  - location: 对象，包含 longitude, latitude, altitude
  - type: 字符串，基站类型（macro/micro/pico）
  - status: 字符串，状态（active/inactive/maintenance）

### 需求:天线参数
系统必须为每个基站配置天线参数。

#### 场景:天线参数结构
- **当** 访问基站天线参数
- **那么** 必须返回以下字段：
  - azimuth: 数值，方位角（0-360度）
  - downtilt: 数值，下倾角（0-90度）
  - power: 数值，发射功率（dBm）
  - height: 数值，天线高度（米）
  - pattern: 字符串/数组，天线方向图（可选）

### 需求:容量模型
系统必须定义基站的容量能力模型。

#### 场景:容量参数结构
- **当** 访问基站容量信息
- **那么** 必须返回以下字段：
  - prb_pool: 整数，PRB资源池大小
  - max_users: 整数，最大同时接入用户数
  - bandwidth: 数值，带宽（MHz）
  - throughput_curve: 数组，吞吐量-负载曲线点
  - carrier_frequency: 数值，载波频率（GHz）

### 需求:状态监控
系统必须提供基站的实时状态监控数据（用假数据填充）。

#### 场景:状态参数结构
- **当** 访问基站状态
- **那么** 必须返回以下字段：
  - load: 数值，实时负载（0-1）
  - active_users: 整数，当前活跃用户
  - alarms: 数组，告警信息列表
  - kpi: 对象，包含：
    - rsrp: 数值，参考信号接收功率（dBm）
    - sinr: 数值，信干噪比（dB）
    - packet_loss: 数值，丢包率（0-1）
    - latency: 数值，延迟（ms）

### 需求:空间特征
系统必须预留空间特征接口（卫星图像、POI），用假数据填充。

#### 场景:空间特征结构
- **当** 访问基站空间特征
- **那么** 必须返回以下字段：
  - satellite_image: 对象，包含：
    - url: 字符串，图像URL（可选）
    - features: 数组，图像特征向量（128维假数据）
  - poi: 对象，包含：
    - nearby: 数组，附近POI列表（类型、距离）
    - features: 数组，POI特征向量（64维假数据）
  - building_density: 数值，建筑密度（0-1，假数据）
  - population_density: 数值，人口密度（人/km²，假数据）

### 需求:数据兼容性
扩展后的基站数据必须保持向后兼容。

#### 场景:现有API兼容
- **当** 现有API访问基站数据
- **那么** 必须正常返回原有的 id 和 loc 字段
- **并且** 新增字段不应影响现有功能
